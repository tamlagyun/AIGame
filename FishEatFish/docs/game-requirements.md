# 《鲫鱼吃鲤鱼》游戏需求基线

需求版本：REQ-0.1  
状态：已确认  
基线日期：2026-07-13

## 产品定位

游戏是一款横屏 2D 海洋大世界单机 PvE 游戏。玩家初始控制一条小鲫鱼，在有限但较大的海域中探索，与 AI 鱼类相互攻击并成长。

## 核心玩法

- 玩家使用虚拟摇杆进行二维游动。
- 初始普通攻击为面向当前朝向的短距离撕咬。
- 初始技能为带冷却的冲刺撕咬。
- 不同技能对应不同撕咬动作标识，供后续动画系统绑定。
- 击败其他鱼后获得经验；升级提高生命上限并回复部分生命。
- 其他鱼由 AI 控制，可攻击玩家或其他鱼。

## 世界与镜头

- 首版采用有限大海域，后续可升级为分区流式加载。
- 使用 2D 正交镜头跟随玩家。
- 鱼身体使用圆形或胶囊形轻量命中判定，核心结算不依赖完整刚体模拟。

## UI 与输入

- 横屏设计分辨率为 1280 × 720。
- 左侧预留虚拟摇杆，右侧预留普通攻击和初始技能按钮。
- UI 必须适配安全区、刘海屏、平板和折叠屏。
- 正式 UI 和美术禁止使用 SVG 或程序化图形拼凑。

## 平台范围

- 微信小游戏
- 抖音小游戏
- Android
- iOS
- HarmonyOS

本阶段不接入平台 SDK，只建立统一接口和构建入口。正式导出仍需要各平台 SDK、AppID、签名和工具链；iOS 原生工程需要 macOS。

## 性能基线

- 主流设备目标 60 FPS，中低端设备不得低于 30 FPS。
- 后续实现必须使用对象池、视野外休眠和分区激活控制鱼群成本。
- 正式资源应使用图集和平台适配纹理，小游戏资源需支持首包与远程包拆分。
- 第一版性能预算暂定：最多 30 条活动鱼、屏幕内最多 16 条完整更新鱼；实际数值须通过真机压测修订。

## 本阶段范围

包含工程骨架、场景层级、核心类型、配置样例、存档契约、平台接口、构建入口、测试、需求和美术档案。

不包含可玩战斗、正式 UI、美术、音频、商业 SDK、签名安装包和在线多人功能。
