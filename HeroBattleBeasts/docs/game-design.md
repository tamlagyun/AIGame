# HeroBattleBeasts 游戏设计

## 1. 项目方向

`HeroBattleBeasts` 是一个卡通 2D 横版动作游戏原型。第一版可玩版本需要融合两类体验：

- 魂斗罗式跑跳射击：快速移动、跳跃、射击、敌人波次、武器强化。
- 冒险岛式平台冒险：多层平台、可爱怪物、金币收集、轻探索、明确的关卡目标。

第一版目标是小而可玩，便于后续扩展。

## 2. 技术路线

正式工程路线采用 Cocos Creator 3.8.x + TypeScript。

原因：

- 后续需要发布到 Android、iOS、微信小游戏、抖音小游戏、鸿蒙平台。
- Cocos Creator 对 2D、动画、音频、UI、资源管理和多平台发布更合适。
- TypeScript 便于拆分核心玩法逻辑，并为后续测试和维护提供类型约束。

早期可以使用编辑器预览和 Web 预览快速验证手感，但正式结构应按 Cocos 项目组织。

## 3. 第一版可玩范围

第一版只做一个短关卡，包含以下行为：

- 玩家可以左右移动。
- 玩家可以跳跃并落在平台上。
- 玩家可以水平射击。
- 基础敌人会巡逻或靠近玩家。
- 子弹可以击败敌人。
- 敌人接触会伤害玩家。
- 金币或宝石可以被收集。
- 武器强化道具可以临时提升射击能力。
- 击败足够敌人或到达出口后通关。
- 生命值耗尽后显示失败和重试状态。

该范围刻意保持克制，用于先验证“横版平台 + 跑跳射击 + 收集通关”的核心闭环。

## 4. 卡通视觉方向

第一版视觉主题建议为“明亮怪兽森林”：

- 主角：小型卡通冒险者，使用玩具感能量枪。
- 敌人：可爱的怪兽，例如圆形史莱姆、带角小怪、飞行虫。
- 关卡：草地平台、树根、木桥、柔和云朵。
- 道具：闪亮金币、彩色能量宝石、发光武器徽章。
- 整体气质：明亮、清晰、轻快，但动作反馈要明确。

该主题同时支持冒险岛式可爱平台感和魂斗罗式战斗可读性。

## 5. 核心循环

1. 玩家进入短关卡。
2. 玩家在平台间跳跃，躲避敌人。
3. 玩家射击敌人并收集金币。
4. 玩家拾取临时武器强化。
5. 玩家清理主要遭遇战并到达出口。
6. 游戏显示胜利、失败或重试状态。

## 6. 操作设计

桌面预览阶段的默认操作：

- `A` / `ArrowLeft`：向左移动。
- `D` / `ArrowRight`：向右移动。
- `W` / `ArrowUp` / `Space`：跳跃。
- `J` / `Z`：射击。
- `R`：胜利或失败后重开。

移动端虚拟摇杆和触摸按钮后续单独确认，不纳入第一版最小玩法实现。

## 7. 主要实体

### 玩家

- 位置、速度、朝向。
- 生命值。
- 是否在地面。
- 当前武器状态。
- 受伤后的短暂无敌时间。

### 敌人

- 位置、尺寸、生命值。
- 移动行为。
- 接触伤害。
- 击败状态。

### 子弹

- 位置和速度。
- 发射者。
- 伤害。
- 存活时间。

### 道具

- 类型：金币、宝石、武器强化、生命恢复。
- 位置。
- 是否已收集。

### 关卡

- 平台。
- 出生点。
- 敌人列表。
- 道具列表。
- 出口区域。

## 8. UI 要求

第一版原型需要显示：

- 玩家生命值。
- 金币数量。
- 敌人击败数量或目标进度。
- 当前武器状态。
- 胜利/失败提示和重开说明。

UI 应保持紧凑，不遮挡主要战斗区域。

## 9. 后续实施步骤建议

1. 创建 Cocos 兼容工程骨架。
2. 建立核心玩法模块和平台抽象接口。
3. 添加关卡、角色、敌人、武器和道具配置。
4. 实现玩家移动、重力、跳跃和平台碰撞。
5. 实现射击、子弹、敌人和命中判定。
6. 实现金币、武器强化、生命值、胜负状态和 HUD。
7. 添加卡通占位美术和基础动效。
8. 为核心逻辑添加可重复运行的验证。

每个步骤都必须先列计划，等待用户确认后再实施。

## 10. 第一版暂不包含

- 多关卡。
- Boss 战。
- 存档。
- 移动端触摸操作。
- 网络功能。
- 最终商业美术。
- 完整音效和音乐。

这些内容应在第一版核心玩法闭环跑通后，再按步骤确认并加入。
